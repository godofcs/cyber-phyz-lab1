## 3. Выбор метрик качества

Для оценки качества моделей выбраны:

- **Precision**
- **Recall**
- **mAP@0.5**
- **mAP@0.5:0.95**

### 3.1. Precision

Precision = TP / (TP + FP). Показывает долю корректных предсказаний среди всех предсказаний модели. Высокий Precision означает, что модель редко выдаёт ложные срабатывания. Для систем мониторинга это важно: ложная тревога приводит к лишним действиям оператора (выезд группы, уведомление и т. п.).

### 3.2. Recall

Recall = TP / (TP + FN). Показывает долю обнаруженных объектов среди всех реальных объектов. Высокий Recall означает, что модель редко пропускает объекты. Для охраны заповедника пропустить браконьерскую активность хуже, чем выдать ложную тревогу.

### 3.3. mAP@0.5

`mAP@0.5` усредняет AP (average precision) по всем классам при пороге IoU = 0.5. Это де-факто стандартная метрика object detection: «нашли ли мы объект и хотя бы примерно правильно его локализовали».

### 3.4. mAP@0.5:0.95

`mAP@0.5:0.95` усредняет AP не только по классам, но и по диапазону порогов IoU от 0.5 до 0.95 с шагом 0.05. Это более жёсткая метрика — она требует, чтобы модель не только нашла объект, но и достаточно точно локализовала его рамкой.

### 3.5. Почему выбраны именно эти метрики

Этот набор является стандартом для object detection (используется в COCO, на нём же основаны таблицы лидеров YOLO/DETR/RT-DETR) и позволяет всесторонне оценить модель:

- `Precision` показывает уровень ложных срабатываний;
- `Recall` показывает уровень пропусков;
- `mAP@0.5` отражает итоговое качество детекции;
- `mAP@0.5:0.95` дополнительно оценивает точность локализации.

Этих четырёх метрик достаточно для корректного сравнения baseline, improved и кастомных моделей.

## 5. Результаты

| Модель | precision | recall | mAP@0.5 | mAP@0.5:0.95 |
|------------|-------|-----------|--------|---------|--------------|
| Baseline (YOLO11n) | 0.9098 | 0.8326 |	0.9160 |	0.7472 |
| Improved (YOLO11s) | 0.9312 | 0.8905 |  0.9440 | 0.7107 |
| Custom default  | 0.5293 | 0.5391 | 0.5446 | 0.3265 |
| Custom improved | 0.5482 | 0.5284 | 0.5551 | 0.2969 |


Лучший результат показала **Improved (YOLO11s)** — `mAP@0.5 = 0.9440` при `Recall = 0.8905`. Это уже практически применимое качество для систем автоматического мониторинга диких животных.

## 6. Выводы

1. **Transfer learning критически важен.** Даже самая лёгкая `YOLO11n` с COCO-весами обходит обучаемую с нуля кастомную архитектуру с разрывом примерно 0.37 по `mAP@0.5`.
2. **Увеличение ёмкости модели работает.** Переход с `YOLO11n` на `YOLO11s` дал +3.1% по `mAP@0.5` и +7.0% по `Recall`.
3. **Аугментации — обоюдоострый инструмент.** Они стабильно поднимают `Recall` и `mAP@0.5`, но могут просаживать `mAP@0.5:0.95`. Чтобы вернуть точную локализацию, можно увеличить число эпох или ослабить геометрические аугментации.
4. **Маленькие модели с нуля плохо учатся на ~1500 изображений.** Для такого датасета transfer learning почти безальтернативен: чтобы поднять кастомную модель до уровня baseline, нужно либо в разы больше данных, либо инициализация осмысленными pre-trained весами.
